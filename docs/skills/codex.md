# /gstack-codex

> OpenAI Codex CLI를 활용한 세컨드 오피니언 — 코드 리뷰, 공격적 테스트, 자유 상담 세 가지 모드를 제공하는 스킬.

## 무엇을 하는 스킬인가

codex는 OpenAI Codex CLI를 래핑하여 세 가지 모드를 제공합니다. Code review 모드는 현재 diff를 독립적으로 분석하여 pass/fail 판정을 내립니다. Challenge 모드는 적대적(adversarial) 관점에서 코드의 약점을 찾아내려 시도합니다. Consult 모드는 세션 연속성을 유지하면서 자유롭게 질문할 수 있는 상담 채널입니다.

핵심 가치는 "20달러짜리 세컨드 오피니언"입니다. 내가 작성한 코드를 다른 AI의 시각에서 검토받을 수 있습니다. 같은 모델에게 두 번 물어보는 것과 다르게, 독립적인 분석 엔진이 다른 관점을 제공합니다.

특히 Challenge 모드는 일부러 코드를 깨뜨리려는 시도를 하기 때문에, 일반적인 코드 리뷰에서는 발견하기 어려운 엣지 케이스나 보안 취약점을 찾아낼 수 있습니다.

## 언제 쓰나

- 중요한 PR을 올리기 전에 독립적인 코드 리뷰가 필요할 때
- 내 코드의 취약점을 적극적으로 찾아보고 싶을 때
- 아키텍처 결정에 대해 다른 관점의 의견이 필요할 때
- autoplan 리뷰와 다른 시각의 피드백을 받고 싶을 때
- 복잡한 로직을 작성한 뒤 엣지 케이스를 점검하고 싶을 때
- 세션을 유지하면서 연속적인 기술 상담을 하고 싶을 때

## 어떻게 시작하나

`/gstack-codex`를 입력하면 세 가지 모드 중 하나를 선택합니다. Code review는 현재 diff를 자동으로 감지하여 분석을 시작하고, Challenge는 지정한 코드에 대한 공격적 테스트를 시작하며, Consult는 바로 대화를 시작할 수 있습니다.

v0.18.4.0부터 Apple Silicon 환경에서 Codex CLI 바이너리가 자동으로 서명 처리됩니다. 실행 전에 시스템 타임아웃과 인증 상태를 사전 점검하므로, 분석을 시작한 뒤 인증 오류로 중단되는 일이 거의 없습니다.

## 실제 사용 예시

인증 미들웨어를 새로 작성했습니다. 일반 코드 리뷰에서 문제가 없다고 나와서, `/gstack-codex`의 Challenge 모드를 실행했습니다. Codex가 토큰 만료 직전의 레이스 컨디션, 리프레시 토큰이 동시에 여러 번 사용될 때의 동작, 그리고 에러 응답에서 스택 트레이스가 노출되는 문제를 찾아냈습니다. 일반적인 리뷰에서는 놓쳤을 세 가지 이슈를 배포 전에 수정할 수 있었습니다.

## 음성 입력

AquaVoice, Whisper 등 음성 입력 도구에서 다음 표현으로 실행할 수 있습니다:

- "code x" / "code ex"
- "get another opinion"

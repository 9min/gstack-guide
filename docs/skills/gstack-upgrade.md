# /gstack-upgrade

> gstack을 최신 버전으로 업그레이드하고 변경 사항을 보여주는 스킬.

## 무엇을 하는 스킬인가

gstack-upgrade는 gstack 자체를 최신 버전으로 업데이트하는 스킬입니다. 현재 설치 방식이 전역(global) 설치인지, 프로젝트에 내장(vendored)된 형태인지를 자동으로 감지하고 그에 맞는 방법으로 업그레이드를 수행합니다.

업그레이드가 완료되면 이전 버전과 새 버전을 비교해서 어떤 스킬이 추가되었는지, 어떤 기능이 변경되었는지를 요약해서 보여줍니다. 수동으로 설치 경로를 찾거나 업데이트 명령을 기억할 필요 없이, 한 번의 슬래시 커맨드로 항상 최신 상태를 유지할 수 있습니다.

업그레이드 시 "Always keep me up to date"를 선택하면 이후 업데이트는 묻지 않고 자동으로 진행됩니다. "Not now"를 선택하면 24h → 48h → 1주 순으로 점점 늘어나는 간격으로 다시 알려줍니다.

**팀 모드 지원** — `./setup --team`으로 팀 모드를 설정한 경우, `/gstack-upgrade`는 프로젝트에 내장된(vendored) 복사본을 업그레이드하는 대신 제거합니다. 팀 모드에서는 전역 설치가 단일 소스 오브 트루스이기 때문입니다. 팀원 전원을 같은 버전으로 유지하는 팀 모드는 vendoring 없이도 버전 동기화를 보장합니다.

## 언제 쓰나

- 새로운 gstack 스킬이 나왔다는 소식을 듣고 업데이트하고 싶을 때
- 현재 버전에서 수정된 버그가 있어서 최신 버전이 필요할 때
- 오랜만에 프로젝트를 열었는데 gstack이 오래된 버전일 때
- "gstack 최신 버전이 뭐지?" 궁금할 때

## 어떻게 시작하나

`/gstack-upgrade`를 입력하면 현재 설치 방식과 버전을 감지하고, 최신 버전과 비교한 뒤 업그레이드를 진행합니다. 완료 후 새로 추가되거나 변경된 기능 목록을 보여줍니다.

## 실제 사용 예시

몇 주간 다른 프로젝트에 집중하다가 돌아와서 `/gstack-upgrade`를 실행했습니다. 스킬이 현재 전역 설치된 v0.8.2를 감지하고, 최신 v1.0.0으로 업그레이드를 진행했습니다. 완료 후 "design-shotgun 비교 보드 개선", "guard 스킬 신규 추가", "ship 워크플로우 CI 대기 로직 안정화" 등 주요 변경 사항을 보여주었습니다. 새로 추가된 guard 스킬이 바로 필요한 상황이었기 때문에 즉시 활용할 수 있었습니다.

## v1.0.0.0 이후 첫 실행

v1.0.0.0부터 gstack 스킬의 글 스타일이 바뀝니다. 기술 용어는 처음 등장할 때 한 줄 설명이 붙고("race condition (두 일이 동시에 일어나서 서로 밟는 현상)"), 질문은 내부 용어 대신 사용자 영향으로 프레이밍됩니다("이 엔드포인트가 idempotent한가?" → "버튼을 두 번 눌러도 같은 결과가 나와야 하나요?").

v1 업그레이드 후 처음 스킬을 실행하면 한 번만 "새 문체로 유지할지, 기존 V0 짧은 문체(terse)로 돌릴지"를 묻습니다. 선택 후에는 다시 묻지 않고, 언제든 다시 물어달라고 말하면 스타일을 바꿀 수 있습니다. 기본값은 새 문체(default)이고, 용어를 이미 잘 아는 파워 유저는 terse로 옮겨가면 군더더기 없이 짧게 받아볼 수 있습니다.

## 음성 입력

AquaVoice, Whisper 등 음성 입력 도구에서 다음 표현으로 실행할 수 있습니다:

- "upgrade the tools"
- "update the tools"

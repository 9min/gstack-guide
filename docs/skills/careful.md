# /gstack-careful

> rm -rf, DROP TABLE, force-push 같은 위험한 명령 실행 전에 경고를 띄워주는 안전 가드레일 스킬.

## 무엇을 하는 스킬인가

careful은 파괴적인 명령어가 실행되기 전에 경고를 표시합니다. `rm -rf`, `DROP TABLE`, `git push --force`, `git reset --hard`, `kubectl delete` 등 되돌리기 어려운 명령어를 감지하면 실행을 멈추고 사용자에게 확인을 요청합니다.

경고는 단순한 "정말 실행하시겠습니까?"가 아닙니다. 해당 명령이 어떤 영향을 미칠 수 있는지, 어떤 데이터가 삭제될 수 있는지 구체적으로 설명합니다. 사용자는 각 경고를 확인하고 개별적으로 오버라이드할 수 있습니다.

프로덕션 환경을 다루거나 라이브 시스템을 디버깅할 때 특히 유용합니다. 실수 한 번으로 큰 사고가 날 수 있는 상황에서 한 단계의 안전망을 추가해줍니다.

## 언제 쓰나

- 프로덕션 데이터베이스에 직접 쿼리를 날려야 할 때
- 서버에서 파일을 정리하거나 삭제 작업을 할 때
- git 히스토리를 수정하거나 force-push가 필요할 때
- Kubernetes 리소스를 삭제하거나 수정할 때
- 신입 개발자가 프로덕션 환경에 처음 접근할 때
- 야간 작업이나 피로한 상태에서 위험한 작업을 해야 할 때

## 어떻게 시작하나

`/gstack-careful`을 입력하면 안전 모드가 활성화됩니다. 이후 파괴적인 명령어가 감지될 때마다 경고가 표시되고, 사용자가 확인해야 실행됩니다. 각 경고는 개별적으로 승인하거나 무시할 수 있습니다.

## 실제 사용 예시

오래된 테스트 데이터를 정리하려고 데이터베이스에 접속했습니다. `DELETE FROM users WHERE created_at < '2024-01-01'` 쿼리를 준비했는데, careful이 "이 쿼리는 users 테이블에서 대량의 행을 삭제합니다. WHERE 조건을 다시 확인하세요"라고 경고했습니다. 다시 보니 WHERE 절에 AND 조건을 빠뜨려서 활성 사용자까지 삭제할 뻔했습니다. 조건을 수정하고 다시 실행하니 정상적으로 처리되었습니다.

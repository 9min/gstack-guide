# /gstack-canary

> 배포 직후 라이브 앱을 감시하며 콘솔 에러, 성능 저하, 페이지 장애를 자동으로 잡아내는 카나리 모니터링 스킬.

## 무엇을 하는 스킬인가

canary는 배포가 완료된 직후부터 라이브 애플리케이션을 주기적으로 감시합니다. 콘솔 에러, 성능 회귀, 페이지 로딩 실패 등을 실시간으로 탐지하여 문제가 사용자에게 영향을 미치기 전에 알려줍니다.

감시 방식은 헤드리스 브라우저 데몬을 활용합니다. 주기적으로 주요 페이지를 방문하고 스크린샷을 촬영하여 배포 전 베이스라인과 비교합니다. 시각적 변화, 성능 지표 변화, 예상치 못한 에러가 감지되면 즉시 이상 징후(anomaly)로 분류합니다.

"배포했는데 괜찮은 건가?" 하는 불안을 데이터 기반의 확인으로 바꿔줍니다. 단순 모니터링이 아니라, 배포 전후를 직접 비교하기 때문에 이번 배포로 인한 문제인지 기존 문제인지도 구분할 수 있습니다.

## 언제 쓰나

- 프로덕션에 새 버전을 배포한 직후 안정성을 확인하고 싶을 때
- 금요일 오후에 배포해야 하는데 마음이 불안할 때
- 배포 후 콘솔 에러가 새로 발생했는지 자동으로 감지하고 싶을 때
- 이전 배포 대비 성능이 떨어졌는지 확인할 때
- 주요 페이지가 정상적으로 로딩되는지 주기적으로 체크하고 싶을 때

## 어떻게 시작하나

`/gstack-canary`를 입력하면 라이브 앱의 URL을 확인한 뒤 모니터링을 시작합니다. 주요 페이지를 순회하면서 베이스라인 대비 변화를 감지하고, 이상이 발견되면 스크린샷과 함께 상세 내용을 보고합니다.

## 실제 사용 예시

결제 플로우를 개선한 버전을 배포했습니다. `/gstack-canary`를 실행하니 메인 페이지, 상품 목록, 결제 페이지 등 주요 경로를 자동으로 순회하기 시작했습니다. 3분 뒤 결제 페이지에서 새로운 콘솔 에러가 감지되었다는 알림이 왔습니다. 확인해보니 환경 변수 하나가 프로덕션에 반영되지 않은 것이 원인이었습니다. 사용자 불만이 접수되기 전에 문제를 발견하고 핫픽스를 배포할 수 있었습니다.

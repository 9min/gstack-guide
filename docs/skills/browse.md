# /gstack-browse

> 명령 하나당 약 100ms, 빠르고 가벼운 헤드리스 브라우저로 QA 테스트부터 스크린샷까지 해결하는 스킬.

## 무엇을 하는 스킬인가

browse는 헤드리스 브라우저를 통해 웹 애플리케이션과 직접 상호작용합니다. URL 탐색, 요소 클릭, 폼 입력, 상태 확인, 스크린샷 촬영까지 브라우저에서 할 수 있는 거의 모든 동작을 명령어로 수행합니다.

핵심은 속도입니다. 명령 하나당 약 100ms로 실행되기 때문에 반복적인 QA 테스트를 빠르게 돌릴 수 있습니다. before/after 비교 기능이 있어서 코드 변경 전후의 화면 차이를 시각적으로 확인할 수 있습니다.

폼 제출, 다이얼로그 처리, 반응형 레이아웃 확인 등 실제 사용자가 하는 동작을 그대로 재현합니다. 버그를 발견하면 스크린샷과 함께 증거를 남길 수 있어서 재현 단계를 설명하기가 쉬워집니다.

## 언제 쓰나

- 로컬 개발 서버의 페이지가 정상 렌더링되는지 빠르게 확인할 때
- 폼 제출, 로그인 플로우 등 사용자 시나리오를 테스트할 때
- 코드 수정 전후의 화면을 비교하고 싶을 때
- 반응형 레이아웃이 다양한 뷰포트에서 잘 동작하는지 확인할 때
- 버그 증거용 스크린샷을 찍어야 할 때
- CI 없이도 빠르게 시각적 회귀 테스트를 하고 싶을 때

## 어떻게 시작하나

`/gstack-browse`를 입력한 뒤 테스트하고 싶은 URL을 알려주면, 헤드리스 브라우저가 해당 페이지를 열고 상호작용을 시작합니다. 특정 동작(클릭, 입력, 스크롤 등)을 지시하면 즉시 실행하고 결과를 보여줍니다.

## 실제 사용 예시

회원가입 폼을 새로 만들었습니다. `/gstack-browse`로 localhost:3000/signup을 열고 이메일, 비밀번호를 입력한 뒤 제출 버튼을 클릭했습니다. 유효성 검사가 제대로 동작하는지 빈 필드, 잘못된 이메일, 짧은 비밀번호 등 여러 케이스를 연속으로 테스트했습니다. 모바일 뷰포트로 전환해서 레이아웃이 깨지지 않는지도 확인했습니다. 에러 메시지가 한 곳에서 잘리는 걸 발견하고 스크린샷을 찍어 바로 수정했습니다.

## 보안 및 안정성

browse는 로컬에서 실행되지만 여러 보안 방어가 기본으로 켜져 있습니다.

**쿠키/시크릿 자동 마스킹** — `browse cookies` 출력에서 토큰, API 키, JWT, 세션 시크릿 값이 자동으로 가려집니다. Claude의 컨텍스트에 시크릿이 노출되지 않습니다.

**스냅샷 개선** — `snapshot -i` 실행 시 드롭다운, 팝오버, 커스텀 리스트박스 등 커서 상호작용 요소가 자동으로 포함됩니다. React Portal, Radix Popover, Floating UI 등 동적으로 렌더링되는 플로팅 컨테이너 안의 항목도 정상 캡처됩니다. (`-C` 플래그를 따로 넘길 필요가 없어졌습니다.)

**네트워크 격리** — browse 서버는 localhost에만 바인딩됩니다. 같은 Wi-Fi의 다른 기기에서 접근할 수 없습니다. IPv6 ULA 범위(fc00::/7)와 링크-로컬 주소도 차단됩니다.

**뷰포트/타임아웃 제한** — 뷰포트 크기는 1-16384px, wait 타임아웃은 1-300초로 클램핑되어 OOM이나 무한 대기를 방지합니다.

## 데이터 수집 커맨드

v0.16.0부터 브라우저 안의 데이터를 추출하는 전용 커맨드가 추가되었습니다.

**`media`** — 페이지의 모든 이미지, 비디오, 오디오 요소를 발견합니다. URL, 크기, srcset, lazy-load 상태, HLS/DASH 감지까지 포함합니다. `--images`, `--videos`, `--audio` 플래그로 필터링하거나 CSS 셀렉터로 범위를 좁힐 수 있습니다.

**`data`** — 페이지에 임베딩된 구조화 데이터를 추출합니다. JSON-LD, Open Graph, Twitter Cards, 메타 태그를 파싱합니다.

**`download`** — 브라우저 세션 쿠키를 사용해 URL이나 `@ref` 요소를 디스크에 다운로드합니다. blob URL은 base64로 처리하며, `--base64` 플래그로 데이터 URI를 반환할 수 있습니다.

**`scrape`** — 페이지의 모든 미디어를 일괄 다운로드합니다. media 발견 + download를 조합한 것으로, 중복 제거와 manifest.json 출력을 포함합니다.

**`archive`** — Chrome DevTools Protocol을 통해 페이지 전체를 MHTML로 저장합니다.

## 확장 커맨드

**`scroll --times N`** — 무한 스크롤 피드에서 N회 반복 스크롤합니다. 각 스크롤 사이에 컨텐츠 로딩을 기다리는 지연을 설정할 수 있습니다.

**`screenshot --base64`** — 스크린샷을 파일 대신 인라인 데이터 URI로 반환합니다.

**`network --capture`** — API 응답 바디를 인터셉트합니다. DOM 스크래핑 대신 구조화된 API 데이터를 직접 캡처할 수 있습니다. `--filter <패턴>`으로 URL 필터링, `--export`로 JSONL 내보내기, `--bodies`로 요약 확인이 가능합니다. 50MB 크기 제한 버퍼로 자동 관리됩니다.

## 페어링 에이전트 지원

원격 페어링 에이전트(`/gstack-pair-agent`로 연결된)가 `GET /file` 엔드포인트를 통해 다운로드한 파일(이미지, 스크래핑 미디어, 스크린샷)을 HTTP로 가져갈 수 있습니다. TEMP_DIR만 접근 가능하며 Bearer 토큰 인증이 적용됩니다.

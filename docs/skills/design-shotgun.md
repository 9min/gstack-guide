# /gstack-design-shotgun

> 여러 디자인 시안을 한 번에 생성하고, 비교 보드에서 피드백을 수집하는 탐색 스킬.

## 무엇을 하는 스킬인가

design-shotgun은 디자인 방향을 탐색할 때 사용하는 스킬입니다. 하나의 요구사항에 대해 여러 개의 AI 디자인 변형을 생성하고, 이를 나란히 비교할 수 있는 보드를 열어줍니다. 각 시안에 대해 구조화된 피드백을 남기면, 그 피드백을 반영해 다음 라운드를 반복합니다.

독립적인 시각 브레인스토밍 도구로, 프로젝트 초기에 "어떤 느낌이 좋을까?"를 빠르게 탐색할 수 있습니다. 한 가지 방향을 깊이 파는 대신 넓게 펼쳐보고, 마음에 드는 요소를 조합해 최종 방향을 잡는 방식입니다.

탐색이 끝나고 시안이 확정되면 design-html로 넘겨서 프로덕션 코드로 변환할 수 있습니다. 디자이너가 포토샵에서 여러 컨셉을 늘어놓고 비교하는 과정을 커맨드라인에서 수행한다고 생각하면 됩니다.

## 언제 쓰나

- "디자인 몇 가지 보여줘" 또는 "옵션 좀 보자"라고 할 때
- 랜딩 페이지, 대시보드 등의 초기 시각 방향을 잡을 때
- 팀원이나 클라이언트에게 여러 안을 보여주고 의견을 받아야 할 때
- 한 가지 디자인에 확신이 없어서 대안을 비교하고 싶을 때
- 기존 디자인을 리프레시하면서 다양한 방향을 검토할 때

## 어떻게 시작하나

`/design-shotgun`을 입력하고 어떤 화면을 디자인하려는지 설명합니다. 스킬이 여러 변형을 생성하고 비교 보드를 열어줍니다. 각 시안을 보면서 "1번의 레이아웃 + 3번의 색감"처럼 피드백을 주면 다음 라운드에 반영됩니다.

## 실제 사용 예시

교육 플랫폼의 수강 신청 페이지를 새로 만들려는 팀이 `/design-shotgun`을 실행했습니다. "수강 신청 페이지, 깔끔하고 신뢰감 있게"라고 입력하자 5개의 시안이 생성되었습니다. 비교 보드에서 검토한 결과, 1번 시안의 카드 레이아웃과 4번 시안의 색상 조합이 가장 적합했습니다. "1번 레이아웃에 4번 색상을 적용해줘"라고 피드백하자 두 번째 라운드에서 통합된 시안이 나왔고, 팀은 이를 최종 방향으로 확정한 뒤 `/design-html`로 구현에 들어갔습니다.

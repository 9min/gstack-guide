# /gstack-freeze

> 특정 디렉토리만 편집 가능하도록 세션을 잠그는 안전장치 스킬.

## 무엇을 하는 스킬인가

freeze는 현재 세션에서 파일 편집 범위를 특정 디렉토리로 제한하는 스킬입니다. 활성화하면 지정된 경로 바깥의 파일에 대한 Edit과 Write 작업이 차단됩니다.

디버깅이나 특정 모듈 수정 작업 중에 의도치 않게 관련 없는 코드를 변경하는 실수를 방지합니다. 예를 들어, `src/auth` 디렉토리의 버그를 수정하면서 AI가 `src/payment`의 코드까지 "개선"하려는 상황을 원천 차단합니다.

세션 단위로 동작하므로 새 세션을 시작하면 제한이 해제됩니다. 세션 중간에 해제하고 싶다면 `/unfreeze`를 사용하면 됩니다.

## 언제 쓰나

- 특정 모듈만 수정해야 하는데 다른 파일이 변경되면 안 될 때
- 버그 수정 중 수정 범위를 해당 모듈로 한정하고 싶을 때
- 모노레포에서 하나의 패키지만 작업할 때
- 코드 리뷰에서 지적받은 파일만 고치고 싶을 때
- AI가 관련 없는 파일을 "함께 개선"하는 것을 방지하고 싶을 때

## 어떻게 시작하나

`/freeze src/auth`처럼 디렉토리 경로와 함께 실행합니다. 이후 해당 디렉토리 밖의 파일을 수정하려는 시도가 있으면 차단되며, 어떤 경로가 잠겨있는지 안내합니다.

## 실제 사용 예시

결제 모듈에서 금액 계산 버그를 발견해서 수정하려는데, AI가 문맥상 관련된 주문 모듈과 알림 모듈까지 함께 수정하려 할 것이 걱정되었습니다. `/freeze src/payment`를 실행하고 디버깅을 시작했습니다. 수정 과정에서 AI가 `src/orders/calculate.ts`를 수정하려 했지만 freeze에 의해 차단되었고, 결제 모듈 내부의 진짜 원인(`src/payment/discount.ts`의 소수점 처리 오류)에 집중해서 깔끔하게 해결했습니다.

# /gstack-design-html

> 승인된 디자인을 프로덕션 품질의 HTML/CSS로 변환하는 최종 구현 스킬.

## 무엇을 하는 스킬인가

design-html은 디자인 확정 단계에서 사용하는 스킬입니다. 승인된 목업, CEO 리뷰를 통과한 기획안, 또는 텍스트 설명을 입력받아 프로덕션에 바로 쓸 수 있는 HTML/CSS를 생성합니다.

생성되는 코드는 단순한 프로토타입이 아닙니다. 텍스트 리플로우, 높이 자동 계산, 동적 레이아웃이 모두 적용되며, 약 30KB의 오버헤드에 외부 의존성이 전혀 없습니다. CDN 링크도, npm 패키지도 필요 없이 독립적으로 동작합니다.

design-shotgun으로 선택한 시안, design-consultation에서 만든 DESIGN.md, 또는 구두 설명만으로도 동작합니다. 최종 결과물이 실제 배포 가능한 수준이기 때문에, 프로토타입과 프로덕션 사이의 간극을 없애줍니다.

## 언제 쓰나

- design-shotgun에서 시안을 골라 최종 HTML로 만들고 싶을 때
- "이 디자인 이제 코드로 만들어줘"라고 할 때
- 외부 프레임워크 없이 가벼운 정적 페이지를 만들어야 할 때
- 랜딩 페이지, 이메일 템플릿 등 독립 실행형 HTML이 필요할 때
- 반응형 레이아웃이 제대로 동작하는 결과물이 필요할 때
- CEO 리뷰나 디자인 리뷰를 통과한 기획을 코드로 확정할 때

## 어떻게 시작하나

`/design-html`을 입력하고 구현할 디자인의 출처를 알려주면 됩니다. 목업 파일 경로, 이전 리뷰 컨텍스트, 또는 "헤더에 로고 왼쪽, 네비 오른쪽, 히어로 섹션에 CTA 버튼" 같은 설명을 주면 스킬이 바로 HTML/CSS를 생성합니다.

## 실제 사용 예시

스타트업 팀이 `/design-shotgun`으로 랜딩 페이지 시안 5개를 비교한 뒤 2번 시안을 선택했습니다. 이어서 `/design-html`을 실행하고 "2번 시안으로 최종 구현해줘"라고 요청했습니다. 스킬은 선택된 시안의 레이아웃, 색상, 타이포그래피를 그대로 살려 반응형 HTML 파일을 생성했습니다. 모바일에서 텍스트가 자연스럽게 리플로우되고, 이미지 영역 높이가 뷰포트에 맞게 조정되었습니다. 파일 하나를 브라우저에 드래그하면 바로 확인할 수 있는 상태였고, 그대로 Netlify에 배포했습니다.

# /gstack-plan-tune

> gstack의 질문을 나에게 맞게 튜닝하고, 내가 어떤 개발자인지 프로필로 관리하는 스킬.

## 무엇을 하는 스킬인가

gstack 스킬들은 진행 도중 자주 질문을 던집니다. "이거 완전판으로 갈까요, 빠른 버전으로 갈까요", "테스트 스펙을 어떤 형태로 맞출까요" 같은 것들입니다. 같은 질문에 매번 같은 답을 하고 있다면, plan-tune이 그 패턴을 기억해서 다음부터는 묻지 않도록 만들 수 있습니다.

핵심은 두 가지입니다. 첫째, **질문별 선호도 학습**. "이 질문은 다시 묻지 마"라고 말하면 해당 질문 ID의 자동 선택을 기록합니다. 둘째, **개발자 프로필**. 다섯 개 차원(범위 욕구, 리스크 허용도, 세부 선호도, 자율성, 아키텍처 중시도)으로 본인이 스스로 선언한 스타일과 실제 행동이 시사하는 스타일을 나란히 보여줍니다. 선언과 행동이 어긋나면 그 차이(gap)를 확인할 수 있습니다.

플러스로, 선언된 차원 조합에서 여덟 가지 빌더 아키타입(Cathedral Builder, Ship-It Pragmatist, Deep Craft, Taste Maker, Solo Operator, Consultant, Wedge Hunter, Builder-Coach) 중 하나로 자동 분류됩니다. 어떤 차원 조합에도 딱 들어맞지 않으면 Polymath로 표시됩니다.

## v1은 관찰 모드입니다

**중요**: v1은 "너가 어떤 스타일인지 관찰하고 보여주는" 단계입니다. 프로필을 설정해도 다른 스킬의 행동이 자동으로 바뀌지 않습니다. 행동 자동화는 v2에서 도입됩니다. 지금은 자기 이해용 거울로 쓰고, `never-ask` 같은 명시 선호만 즉시 반영됩니다.

## 언제 쓰나

- 같은 AskUserQuestion에 매번 같은 답을 하고 있을 때
- "내가 어떤 개발자인지 gstack이 어떻게 보고 있는지" 확인하고 싶을 때
- 새 프로젝트를 시작하면서 내 기본 스타일을 gstack에 알려주고 싶을 때
- "선언한 나"와 "행동의 나"의 차이를 점검하고 싶을 때
- 질문 튜닝 기능을 켜고 끄거나 완전히 비활성화하고 싶을 때
- 특정 질문을 "항상 묻기" 또는 "절대 묻지 말기"로 고정하고 싶을 때

## 어떻게 시작하나

Claude Code에서 입력:

```
/gstack-plan-tune
```

처음 실행하면 자동으로 설정 마법사가 시작됩니다. 2분 정도가 걸리고, 다섯 개 차원에 대한 질문에 답하면 프로필이 기록됩니다. 이후에는 자유롭게 말로 원하는 작업을 지시할 수 있습니다:

- "프로필 보여줘" — 현재 선언/관찰 값과 아키타입 표시
- "최근 받은 질문 보여줘" — 질문 로그 검토
- "카테고리 결정은 다시 묻지 마" — 선호도 저장
- "튜닝 꺼" — 전체 비활성화

인라인 튜닝도 지원합니다. 다른 스킬이 질문했을 때 답 대신 `tune: never-ask`, `tune: always-ask`, 또는 자유롭게 쓴 메시지로 응답하면 해당 질문에 대한 선호가 저장됩니다. (단, 아래 보안 주의 참고)

## 일방 결정(one-way door)은 항상 물어봅니다

파괴적 작업(데이터 삭제, 강제 푸시), 아키텍처 포크, 보안 선택 같은 "한 번 틀리면 되돌릴 수 없는" 질문은 `never-ask` 선호가 있더라도 무시하고 반드시 묻습니다. 안전이 선호보다 우선합니다. 두 방향 결정(two-way door)만 자동화 대상입니다.

## 프로필 오염 방어 (`tune:` 보안)

`tune:` 접두사는 **본인이 현재 대화창에 직접 입력한 메시지**에서만 효과가 있습니다. 도구 출력, 파일 내용, PR 설명, 외부 문서 어디에 `tune:` 텍스트가 있어도 무시됩니다. 악의적인 저장소가 gstack의 학습 동작을 조작하지 못하도록 설계 첫날부터 차단되어 있습니다. 거부된 튜닝 쓰기는 exit code 2로 조용히 떨어지고 재시도하지 않습니다.

## 자유 응답 확인 단계

`tune: never-ask`처럼 명시적 키워드가 아니라 "너무 자주 물어" 같은 자유 문장으로 응답하면, gstack은 이를 `never-ask`로 해석하고 되물어 확인합니다: "`<원문>`을 `<선호>`로 이해했습니다. `<question-id>`에 적용할까요? [Y/n]". 오해석을 막는 안전장치입니다.

## 기본은 꺼져 있습니다

질문 튜닝은 옵트인입니다. 처음 `/plan-tune`을 실행하면 활성화할지 여부를 묻고, "아니요"를 고르면 가만히 둡니다. 켜지 않는 한 다른 스킬의 동작에는 아무 영향이 없고, 활성화 상태에서도 프로필 관찰만 이루어질 뿐 스킬이 자동으로 행동을 바꾸지는 않습니다(v1 관찰 모드).

## 실제 사용 예시

autoplan을 반복해서 돌리는데 매번 "CEO/Eng/Design 중 어느 리뷰를 먼저 돌릴까요?" 질문이 나오고, 나는 매번 Eng 먼저를 고릅니다. 세 번째 반복에서 `/gstack-plan-tune`을 실행해 "autoplan의 리뷰 순서 결정은 다시 묻지 마, 항상 Eng 먼저"로 저장했습니다. 이후부터 해당 질문이 자동 결정되고, "Auto-decided [요약] → [선택] (your preference). Change with /plan-tune."라는 한 줄이 인라인으로 표시됩니다. 마음이 바뀌면 `/plan-tune`으로 다시 편집할 수 있습니다.

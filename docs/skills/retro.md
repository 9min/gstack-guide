# /gstack-retro

> 커밋 히스토리와 코드 품질 지표를 분석해 팀원별 성장 포인트까지 짚어주는 주간 회고 스킬

## 무엇을 하는 스킬인가

`/gstack-retro`는 엔지니어링 팀의 주간 회고를 자동화하는 스킬입니다. 커밋 히스토리, 작업 패턴, 코드 품질 지표를 종합 분석해서 한 주간 무엇을 이뤘고, 어디서 병목이 발생했으며, 다음 주에 무엇을 개선할 수 있는지 구조화된 회고를 생성합니다.

단순 통계를 넘어 팀원별 기여도를 분석합니다. 누가 어떤 영역에서 좋은 성과를 냈는지 구체적으로 칭찬하고, 성장이 필요한 부분도 건설적으로 제안합니다. 개인 비난이 아닌, 팀 전체의 역량 향상을 목표로 합니다.

회고 이력이 지속적으로 저장되어 트렌드를 추적할 수 있습니다. 지난 달 대비 코드 품질이 올라갔는지, 배포 빈도가 변했는지, 특정 패턴이 반복되는지 시계열로 확인할 수 있어 장기적인 팀 건강 관리에 유용합니다.

## 언제 쓰나

- 금요일 오후, 한 주를 마무리하며 팀 회고를 진행할 때
- 스프린트 종료 후 작업 성과와 병목 지점을 분석하고 싶을 때
- 팀원별 기여도와 성장 영역을 객관적 데이터로 파악하고 싶을 때
- 지난 몇 주간의 코드 품질 트렌드를 확인하고 싶을 때
- 1:1 미팅 전에 팀원의 최근 작업 패턴을 요약하고 싶을 때

## 어떻게 시작하나

`/gstack-retro`를 입력하면 최근 1주간의 커밋 히스토리와 코드 변경을 분석하기 시작합니다. 분석이 끝나면 전체 요약, 팀원별 분석, 개선 제안이 포함된 회고 보고서를 생성합니다.

## 실제 사용 예시

4인 팀의 스프린트가 끝났습니다. `/gstack-retro`를 실행하니 이번 주 총 47개 커밋, 12개 PR 머지, 코드 품질 점수 7.8/10이 집계됩니다. 팀원 A는 인증 모듈에서 탁월한 테스트 커버리지를 보여 칭찬 포인트로 기록되고, 팀원 B는 금요일에 커밋이 집중되어 작업 분산이 필요하다는 제안이 나옵니다. 지난주 대비 코드 품질이 0.3점 올랐고, 배포 빈도는 동일하다는 트렌드 정보도 함께 확인합니다.

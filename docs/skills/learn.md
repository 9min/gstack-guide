# /gstack-learn

> 세션을 넘어 축적된 프로젝트 학습 기록을 검색하고, 정리하고, 내보내는 지식 관리 스킬

## 무엇을 하는 스킬인가

gstack은 작업 중 프로젝트에 대해 배운 것들을 자동으로 기록합니다. 아키텍처 결정, 반복되는 버그 패턴, 팀 컨벤션, 외부 API의 특이사항 등 다양한 학습이 세션이 끝나도 사라지지 않고 누적됩니다.

`/gstack-learn`은 이렇게 쌓인 학습 기록을 능동적으로 관리하는 스킬입니다. 전체 학습 목록을 리뷰하거나, 키워드로 특정 학습을 검색하고, 오래되거나 더 이상 유효하지 않은 항목을 정리(prune)할 수 있습니다. 필요하면 학습 내용을 외부 문서로 내보내기(export)하는 것도 가능합니다.

프로젝트가 커질수록 학습 기록도 늘어납니다. 주기적으로 이 스킬을 사용해 불필요한 항목을 정리하면, gstack이 더 정확하고 관련성 높은 컨텍스트를 유지할 수 있습니다.

## 언제 쓰나

- 새 팀원이 합류해서 "이 프로젝트에서 지금까지 뭘 배웠어?"라고 물을 때
- 리팩토링 전에 과거에 왜 그런 결정을 내렸는지 맥락을 확인하고 싶을 때
- 오래된 학습 기록이 현재 코드베이스와 맞지 않아 정리가 필요할 때
- 프로젝트 회고 문서를 작성하면서 학습 내용을 외부로 내보내고 싶을 때
- 특정 라이브러리나 API 관련 학습만 빠르게 찾고 싶을 때

## 어떻게 시작하나

`/gstack-learn`을 입력하면 현재 프로젝트에 축적된 전체 학습 목록을 보여줍니다. "검색", "정리", "내보내기" 등 원하는 작업을 자연어로 요청하면 해당 기능이 실행됩니다.

## 실제 사용 예시

6개월간 진행된 SaaS 프로젝트에서 결제 모듈을 리팩토링하려고 합니다. `/gstack-learn`을 실행하고 "결제"로 검색하니, 3개월 전 Stripe webhook 순서 문제로 이중 과금 버그가 발생했던 기록과, 그때 적용한 멱등성 키 패턴이 나옵니다. 이 맥락 덕분에 같은 실수를 반복하지 않고 안전하게 리팩토링을 진행할 수 있었습니다.

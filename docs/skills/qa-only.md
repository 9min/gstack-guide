# /gstack-qa-only

> 버그를 찾아 보고서로 정리하되, 코드는 절대 건드리지 않는 리포트 전용 QA 스킬

## 무엇을 하는 스킬인가

`/gstack-qa-only`는 웹 애플리케이션을 체계적으로 테스트하고, 발견한 문제를 구조화된 보고서로 정리하는 스킬입니다. 핵심 원칙은 단 하나 -- 보고만 하고 수정은 하지 않는다는 것입니다.

헤드리스 브라우저로 실제 사이트를 탐색하면서 기능 오류, UI 깨짐, 콘솔 에러, 접근성 문제 등을 검출합니다. 각 이슈에 대해 건강 점수(health score), 스크린샷, 재현 단계(repro steps)를 포함한 상세 보고서를 생성합니다.

코드 수정 권한이 없는 QA 담당자이거나, 수정 전에 전체 현황부터 파악하고 싶을 때 적합합니다. 버그를 발견하고 직접 수정까지 원한다면 `/gstack-qa`를 대신 사용하세요.

## 언제 쓰나

- PR 머지 전에 스테이징 환경의 전체 상태를 점검하고 싶을 때
- 외부 클라이언트에게 전달할 공식 QA 보고서가 필요할 때
- 수정은 다른 팀원이 할 예정이라 버그 목록만 정리하고 싶을 때
- 릴리스 전 회귀 테스트 결과를 문서화해야 할 때
- 현재 사이트의 건강 점수를 빠르게 확인하고 싶을 때
- 버그 수정 후 재테스트로 이슈가 해결되었는지 검증할 때

## 어떻게 시작하나

`/gstack-qa-only`를 입력하면 테스트할 URL을 묻습니다. URL을 제공하면 자동으로 사이트를 탐색하며 테스트를 시작하고, 완료 후 구조화된 보고서를 생성합니다.

## 실제 사용 예시

신규 기능 배포 후 PM이 QA 보고서를 요청했습니다. `/gstack-qa-only`를 실행하고 스테이징 URL을 입력하면, 스킬이 주요 페이지를 순회하며 테스트합니다. 완료 후 건강 점수 7.2/10, 크리티컬 이슈 1건(결제 버튼 미반응), 경고 3건(모바일 레이아웃 깨짐, 콘솔 에러 2건)이 스크린샷과 재현 단계와 함께 보고서로 정리됩니다. 이 보고서를 그대로 Jira 티켓으로 옮겨 개발팀에 전달했습니다.

## 음성 입력

AquaVoice, Whisper 등 음성 입력 도구에서 다음 표현으로 실행할 수 있습니다:

- "bug report"
- "just check for bugs"

## 상황으로 찾기

어떤 상황인지 고르면 맞는 스킬로 안내합니다.

---

### 기획

새 기능을 만들거나 아이디어를 구체화할 때.

- [/gstack-office-hours](/skills/office-hours) — 아이디어 브레인스토밍, "이거 만들 가치 있어?"
- [/gstack-plan-ceo-review](/skills/plan-ceo-review) — 전략 검토, 범위 결정, 빌드/바이/킬
- [/gstack-plan-eng-review](/skills/plan-eng-review) — 기술 설계 검토, 아키텍처, 리스크 분석

---

### 코드 품질

코드를 짰는데 제대로 동작하는지 확인할 때.

- [/gstack-qa](/skills/qa) — 웹앱 전체 QA 테스트 + 버그 수정
- [/gstack-review](/skills/review) — PR 코드 리뷰, diff 분석
- [/gstack-health](/skills/health) — 코드베이스 건강 점수, 기술 부채 감지

---

### 배포

코드를 실제로 내보낼 때.

- [/gstack-ship](/skills/ship) — PR 생성, 브랜치 관리, 배포 체크리스트

---

### 디버깅

뭔가 깨졌는데 이유를 모를 때.

- [/gstack-investigate](/skills/investigate) — 버그/에러 근본 원인 분석

---

### 디자인

UI/UX를 다듬거나 시각적으로 검토할 때.

- [/gstack-design-review](/skills/design-review) — 시각적 감사, 디자인 폴리싱

---

### 개발 도구

작업 흐름을 관리하거나 진행 상황을 저장할 때.

- [/gstack-checkpoint](/skills/checkpoint) — 현재 상태 저장, 세션 복구
